# Project Instructions for Codex

## Goal
Create a clean, fast, static personal/home landing page hosted on GitHub Pages.

## Hosting target
- Host with GitHub Pages.
- Prefer plain HTML, CSS, and minimal JavaScript.
- Avoid frameworks, build steps, and package managers unless explicitly requested.
- Use relative paths so the site works on GitHub Pages project URLs and custom domains.

## Site requirements
- Main file: index.html
- CSS file: styles.css
- Optional JS file: script.js only if needed
- Assets go in /assets
- Include responsive mobile-first layout.
- Include good SEO tags, Open Graph tags, title, description, and favicon placeholders.
- Include accessible semantic HTML.
- Keep performance lightweight.

## Design direction
- Modern landing page.
- Strong hero section.
- Clear call to action.
- Sections for bio/about, selected work, services/projects, contact/social links.
- Avoid generic AI-looking gradients unless specifically requested.

## Quality checks
Before finishing:
- Verify index.html links to styles.css correctly.
- Check layout at mobile and desktop widths.
- Check alt text, heading order, and readable contrast.
- Make sure there are no broken local asset paths.
